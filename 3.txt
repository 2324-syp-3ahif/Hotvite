### 3.2.3 View Event
#### 3.2.1 GUI-Design
> ![EventChat](assets/Event.png)

#### 3.2.2 Workflow
> Hier findet man die Liste der Teilnehmer und kann prüfen, ob Freunde dabei sind. Teilnehmer haben hier die Möglichkeit, direkt ihre Teilnahme zu bestätigen oder abzusagen. Selbst nach dem Beitritt des Events, ist es möglich es jederzeit wieder zu verlassen.


## 3.3. Create Event
#### 3.3.1 GUI-Design
> ![CreatEvent](assets/CreatEvents.png)

#### 3.3.2 Workflow
> Events können nur erstellt werden, wenn man angemeldet ist. Bei der Erstellung müssen der Name, das Datum/Uhrzeit und die Art des Events angegeben werden. Zusätzlich besteht die Möglichkeit, eine Beschreibung sowie Anforderungen (z.B. Mindestalter). Ebenso muss der Eintrittspreis angegeben werden. Der Veranstalter kann außerdem festlegen, ob eine Chatfunktion für dieses Event aktiviert werden soll.


## 3.4 Manage Event
#### 3.4.1 Workflow
> Nach der Erstellung des Events können weiterhin Änderungen vorgenommen werden. Der Name sowie die Beschreibung können geändert werden. Sollte das Event verschoben werden, ist es möglich, dies direkt anzupassen. Zusätzlich können Bedingungen hinzugefügt oder entfernt werden, und der Eintrittspreis kann erhöht oder gesenkt werden. Der Veranstalter hat auch die Möglichkeit, die Chatfunktion nachträglich zu aktivieren oder zu deaktivieren.


## 3.5 Event Chat
#### 3.5.1 GUI-Design
> ![ManageRunningEvent](assets/Chat.png)

#### 3.5.2 Workflow
> Sobald der Chat aktiviert ist, können sowohl der Veranstalter als auch die Teilnehmer miteinander chatten. Teilnehmer erhalten Zugriff auf den Chat, sobald sie dem Event beigetreten sind. Nachdem man das Event verlassen hat, bleibt der Chat zwar sichtbar, jedoch ist es einem nicht mehr möglich, Nachrichten zu schreiben.


## 3.6 Profile
### 3.6.1 My Profile
#### 3.6.1.1 GUI-Design
> ![ManageProfile](assets/ManageProfil.png)

#### 3.6.1.2 Workflow
> Nach dem Einloggen kann das Profil angepasst werden, indem man auf sein Profil geht, auf "Bearbeiten" klickt und dann anschließend nach Belieben seinen Namen, die Beschreibung und sein Profilbild ändern. Zusätzlich kann man auch seine Sozial-Media verlinken. Hier kann man alle seine Veranstaltungen und Teilnahmen auf einen Blick sehen. Wenn man auf eines der Events klickt, kommt man zur Übersicht, des jeweiligen Events.   

### 3.6.2 Other Profiles
#### 3.6.2.1 GUI-Design
> ![ManageProfile](assets/Profil.png)

#### 3.6.2.2 Workflow
> Wenn du auf das Profil von anderer klickst, sieht man auf einen Blick alle Events, die diese Person veranstaltet.


## 3.7 Save Events
#### 3.7.1 GUI-Design
> ![SaveEvents](assets/SaveEvents.png)

#### 3.7.2 Workflow
> Wenn man sich nicht sicher ist, ob man auf ein Event wirklich gehen will, kann man sich das Event auch für später speichern, indem man auf der Übersicht des Events auf den Knopf "Interessiert" klickt. Danach findest du das Event unter deinen gespeicherten Events. Bei deinen gespeicherten Events sieht man die wichtigsten Informationen eines Events (Standort, Art Datum und Teilnehmer). Hier kannst du dich auch direkt für das Event anmelden, ohne auf die Übersicht zu wechseln. Man kann aber auch den "Verwerfen" Knopf klicken, um es aus deinen gespeicherten Events zu entfernen.


## 3.8. Notifications
#### 3.8.1 Workflow
> Wenn das Event nachträglich geändert wird, erhalten die Teilnehmer eine Nachricht und werden sofort über Änderungen informiert, wie zum Beispiel eine geänderte Uhrzeit des Events.
